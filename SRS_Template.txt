1. Purpose
	This document outlines the requirements for Group 1's corporate communication application.
        1.1. Scope
			This Document will catalog the user, system, and hardware requirements for Group 1's corporate communication application. It will not document how those requirements are to be implemented.
        1.2. Definitions, Acronyms, Abbreviations
			Client: The user-facing program for interacting with the system. Clients will connect to a singular server.
			Server: The program designed to host the system. Multiple clients connect to this server.
			Administrator: An elevated user with privileges that allow monitoring server chat logs, creating and deleting users.  

        1.3. References
			Use Case Specifications - Section 5
			UML Use Case Diagrams Section 6
			Class Diagrams – Section 7
			Sequence Diagrams – Section 8
        1.4. Overview
			Group 1's corporate communication application is designed to provide rapid wireless communication for all employees of a very large company. Messages sent will be immutable and records of every message sent 
			will be logged and accessible to internal IT users. The application will be interactable through a GUI, and connections will be serviced over TCP/IP.

			Group and direct messaging
